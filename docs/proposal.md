# 📋 專案提案書：RentalHub — 個人房東租賃管理系統 v3

> **一句話描述**：個人房東的一站式租賃管理工具
> **技術棧**：Next.js + Firebase（Firestore + Auth + Storage）
> **提案日期**：2026年5月18日
> **狀態**：Phase 1 完成 ✅ → Phase 2 進行中

---

## 🎯 專案概述

### 核心定位
**「個人房東的數位管家」**

管理多個建案、多間客房、多位房客的完整租賃生命週期。

### 使用情境
- 你擁有多個建案（大樓），每個建案有數間客房不等
- 每間客房有獨立合約、租金、費用計算
- 水電瓦斯用手動輸入度數，系統自動結算
- 修繕清潔派工追蹤
- 所有證據圖片雲端保存

## 🎨 功能規劃

### MVP（Must Have）— 7 個核心功能

#### M1：建案與客房管理
- 建案基本資料（名稱、地址、類型、可出租間數、水號、電號、瓦斯號、管理費、第四台/網路、扣繳方式、Wifi、門禁密碼、環境描述）
- 建案照片（地址輸入完成後產生並儲存固定尺寸 Google Static Maps 清晰地圖圖片；地圖使用較小顯示範圍讓街道名更清楚；管理員可上傳建築物外觀照片，或使用系統從 Google Place 取得的預設照片；預設圖先查地址 Place，若沒有 photos 再查附近地標／街景／大樓名，仍沒有才存空值；外觀照需裁切為適當比例並同時儲存預覽小圖與放大大圖，建案卡片只讀小圖加速更新）
- 建案卡片採窄版 6:9 圖片輪播設計，提供地圖連結、客房資訊按鈕、編輯資料按鈕，不顯示已租/空置/百分比進度；Title 下方提供出租狀態、租金區間、建案類型、國家、城市篩選；預覽圖可點擊放大；建案頁下方不再顯示客房列表
- 客房基本資料（房號、類型、坪數、樓層、租金、設備清單、網路／第四台資訊、其他資訊）
- 客房資訊頁（修改建案房間數量後自動建立缺少的客房資料欄位；頁面開啟預設顯示第一間客房；顯示建案名稱、地址、出租狀態篩選與房號選擇；選擇房號後下方資訊即時切換，管理員可直接修改每個主要欄位；可將目前房間資料套用到同建案其他房間且保留各房房號；室內圖最多 6 張，儲存預覽小圖與放大大圖，支援左右滑動與點擊放大）
- 客房狀態（已出租/空置/維修中/清潔中）
- 建案儀表板（一眼看所有客房狀態）
- 客房照片（屋況紀錄，含時間戳）

#### M2：房客管理
- 房客基本資料（姓名、電話、Email、LINE ID）
- 緊急聯絡人
- 身分證字號（加密儲存）
- 職業與收入區間
- 房客聯絡紀錄（通話、訊息、會面）
- 房客評分（繳租準時度、房屋維護、溝通態度）

#### M3：合約管理
- 合約建立（模板 + 自訂條款）
- 合約內容：租期、租金、押金、費用分攤方式
- 合約到期提醒（30天/7天/1天前，LINE + Email）
- 一鍵續約
- 合約版本管理
- 合約文件掃描儲存

#### M4：租金管理
- 每月租金紀錄（應收/實收/差額）
- 收款方式（轉帳/現金/混和）
- 收款日期紀錄
- 逾期自動提醒
- 租金調整歷史
- 月/季/年報表

#### M5：智慧費用計算（水電瓦斯）
- **每月手動輸入總度數**（電表、水表、瓦斯表）
- **自動計算公式**：
  - 水費：依坪數比例分攤
  - 電費：依台數/人數比例分攤
  - 瓦斯：依人數比例分攤
  - 第四台：均攤
  - WiFi：均攤
- **每月自動生成費用明細單**
- **一鍵分享給房客**（PDF / LINE）
- **歷史費用趨勢圖**

#### M6：修繕清潔派工
- 修繕請求建立（房東自行建立 or 房客回報）
- 派工單（指派廠商/自行處理）
- 進度追蹤（待處理/進行中/已完成/已驗收）
- 費用紀錄（材料費 + 工資）
- 修繕照片（前後對比）
- 清潔排程改由獨立行事曆頁管理，不放在修繕派工頁內
- 廠商資料庫（聯絡方式、報價、評價）

#### M7：證據圖片保存
- 屋況照片（入住前、退租後、定期檢查）
- 溝通紀錄截圖
- 收據/發票掃描
- 合約文件
- 分類標籤 + 時間戳
- 雲端儲存（Firebase Storage）
- 快速搜尋

│  │  管理    │ │  管理    │ │  管理    │ │  管理    │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                │
│  │證據保存  │ │通知中心  │ │報表統計  │                │
│  └──────────┘ └──────────┘ └──────────┘                │
└──────────────────────┬──────────────────────────────────┘
                       │
        ┌──────────────┼──────────────┐
        │              │              │
   ┌────┴────┐   ┌─────┴─────┐  ┌────┴────┐
   │Firestore│   │Firebase   │  │Firebase │
   │(資料庫) │   │Auth       │  │Storage  │
   │         │   │(登入認證) │  │(圖片)   │
   └─────────┘   └───────────┘  └─────────┘

### Firebase 服務使用（3 個）

| 服務 | 用途 | 說明 |
|------|------|------|
| **Firestore** | 主要資料庫 | 所有資料儲存（建案、客房、房客、合約、租金、費用、修繕） |
| **Firebase Authentication** | 登入認證 | Email/Phone 登入，支援多裝置 |
| **Firebase Storage** | 圖片/文件儲存 | 屋況照片、合約掃描、收據、證據圖片 |

> ⚠️ **不使用 Cloud Functions**：所有後端邏輯放在 Next.js API Routes 中處理（費用計算、通知觸發等）

### Firestore 資料結構

```
/users/{userId}                              → 房東帳號
/users/{userId}/projects/{projectId}         → 建案
/users/{userId}/projects/{projectId}/rooms/{roomId}  → 客房
/users/{userId}/tenants/{tenantId}           → 房客
/users/{userId}/contracts/{contractId}       → 合約
/users/{userId}/rentRecords/{recordId}       → 租金紀錄
/users/{userId}/utilityBills/{billId}        → 水電瓦斯費用
/users/{userId}/maintenanceTasks/{taskId}    → 修繕派工
/users/{userId}/cleaningSchedules/{scheduleId} → 清潔排程
/users/{userId}/evidenceFiles/{fileId}       → 證據文件
/users/{userId}/vendors/{vendorId}           → 廠商
/users/{userId}/notifications/{notificationId} → 通知
```

### 智慧費用計算邏輯

```
每月結算流程：
1. 房東手動輸入當月總度數（電表、水表、瓦斯表）
2. 系統自動抓取各客房上期度數
3. 計算各客房本期用量 = 本期度數 - 上期度數
4. 依分攤規則計算各房客應繳金額
5. 加上固定費用（第四台、WiFi 均攤）
6. 生成費用明細單
7. 發送通知給房客

分攤規則：
- 水費：依坪數比例
  房客水費 = 總水費 × (房客坪數 / 總坪數)
- 電費：依台數比例（冷氣台數）
  房客電費 = 總電費 × (房客台數 / 總台數)
- 瓦斯：依人數比例
  房客瓦斯費 = 總瓦斯費 × (房客人數 / 總人數)
- 第四台：均攤
  房客第四台 = 總第四台費 / 總人數
- WiFi：均攤
  房客WiFi = 總WiFi費 / 總人數
```
